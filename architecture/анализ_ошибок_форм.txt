Анализ ошибок в генераторе форм

1. Проблема: OptionGroup (Радиокнопки) не редактируются в конструкторе
   - Описание: Невозможно задать начальные значения (например, "Муж"/"Жен") для контрола OptionGroup в визуальном редакторе.
   - Причина: В текущей реализации, вероятно, отсутствует обработка свойств "Items" или "Options" для контрола OptionGroup в панели свойств (Property Grid). Контрол отображается, но его внутренние значения (список опций) не привязаны к полю в таблице данных.
   - Ожидаемое поведение: При добавлении OptionGroup на форму, пользователь должен иметь возможность:
     a) Указать список значений (Labels) для радиокнопок.
     b) Привязать контрол к полю в таблице (DataField).
     c) Система должна автоматически подставлять значение в зависимости от данных в строке (например, если в поле "Пол" значение "Жен", то радиокнопка "Жен" должна быть выбрана).

2. Проблема: Отсутствие привязки к источнику данных (DataBinding) для OptionGroup
   - Описание: Контрол OptionGroup не "видит" данные из таблицы. Если в таблице есть строка со значением "Жен", OptionGroup не реагирует на это.
   - Причина: Отсутствует логика двустороннего связывания (Two-Way Binding) между значением радиокнопки (Value) и данными в поле таблицы (Field).
   - Ожидаемое поведение: При привязке к полю, контрол должен считывать текущее значение из данных и отображать его. При изменении пользователем выбора, значение должно записываться обратно в источник данных.

3. Проблема: Контролы активны в режиме генерации форм (Design Mode)
   - Описание: В режиме визуального редактирования (Design Mode) контролы ведут себя как в рантайме (Run Mode). Можно вводить текст в TextBox, выбирать даты в DatePicker, кликать по кнопкам.
   - Причина: Не реализован или некорректно работает механизм "Design Mode". Контролы не проверяют текущее состояние редактора (режим дизайна vs режим выполнения).
   - Ожидаемое поведение:
     a) В режиме генерации (Design Mode) все интерактивные контролы должны быть "заблокированы" для ввода.
     b) Должна быть возможность только выделять контролы, перемещать их, изменять размеры и свойства в Property Grid.
     c) Ввод данных, выбор дат, клики - должны быть доступны ТОЛЬКО в режиме выполнения (Run Mode) или при предпросмотре (Preview).

4. Общие замечания по UX/UI редактора
   - Необходимо четко разделить визуально режимы (например, разный курсор, или блокировка фона).
   - Свойства контролов должны быть контекстно-зависимыми (например, для OptionGroup показывать список опций, а не только Text).

Вывод:
Текущая реализация генератора форм не соответствует требованиям визуального конструктора. Необходимо доработать механизм привязки данных для специфических контролов (OptionGroup) и внедрить полноценную поддержку "Design Mode" для предотвращения случайного ввода данных при проектировании интерфейса.
