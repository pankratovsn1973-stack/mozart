Приоритет 5. DataBinding и передача данных на L2
Как должно работать:
    1. Форма получает entity_instance_id и entity_alias при открытии.
    2. Загружает данные через L2 (вызов get_item).
    3. Заполняет контролы (DataBinding).
    4. При изменении данных — помечает контролы как dirty.
    5. При сохранении — собирает изменённые данные, отправляет на L2 через save_item.
    6. L2 выполняет транзакцию (создаёт новую версию или обновляет).
Что нужно в конфигураторе:
    • Для каждого контрола, привязанного к полю сущности, в свойствах указывать field (имя поля в ext-таблице).
    • В форме хранить список привязок: {control_alias: field_name}.

Как форма загружает данные
python
# В методе on_open формы
def on_open(self, params):
    entity_alias = params.get('entity_alias')
    instance_id = params.get('instance_id')
    
    if instance_id:
        # Форма просит DataBridge дать данные
        data = self.data_bridge.get_item(entity_alias, instance_id)
        # DataBridge сам шифрует, отправляет, дешифрует ответ
        # Форма получает готовый словарь
        
        # Заполняем контролы
        for control_alias, field_name in self.bindings.items():
            control = self.get_control(control_alias)
            if control and field_name in data:
                control.value = data[field_name]

Как форма сохраняет данные
python
# При нажатии кнопки "Сохранить"
def on_save(self):
    # Собираем данные из контролов
    data = {}
    for control_alias, field_name in self.bindings.items():
        control = self.get_control(control_alias)
        if control:
            data[field_name] = control.value
    
    # Форма просит DataBridge сохранить
    result = self.data_bridge.save_item(
        entity_alias=self.entity_alias,
        instance_id=self.instance_id,
        data=data,
        version_id=self.version_id
    )
    # DataBridge сам шифрует, отправляет, получает ответ
    
    if result.success:
        self.version_id = result.new_version_id
        # Обновляем original_value у всех контролов
        self.reset_dirty()

Что нужно в конфигураторе для DataBinding
При настройке формы в конфигураторе, для каждого контрола (кроме кнопок, панелей и т.д.) указываем:
Свойство
Назначение
binding_field
Имя поля в ext-таблице (например, cname, clastname)
binding_entity
Алиас сущности (если отличается от основной формы)
Пример для Reference-контрола:
json
{
    "calias": "person_id",
    "binding_field": "person_id",
    "binding_entity": "person",
    "display_field": "cname"
}

Итог
    • Форма не знает про L2 и шифрование.
    • Форма знает только L3_DataBridge.
    • L3_DataBridge — единственный, кто умеет отправлять на L2 и шифровать.
    • L2_DataBridge — единственный, кто принимает от L3.
    • CryptoEngine — единственный, кто умеет шифровать/дешифровать.
